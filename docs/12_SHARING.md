# Shareable Artifacts

Three artifact kinds exist. Every artifact is a row in `artifact`, a file in object storage, and a short_code routable URL.

## 1. Single clip share

The simplest share. From any what-if clip card the user can hit "Share."

**What ships:**
- The clip MP4.
- A landing page at `https://whatif.app/c/<short_code>`:
  - The clip plays.
  - Banner: "[User] asked 'what if [text]'" with the live match metadata as small print.
  - "What is this?" expander explains the system in one paragraph.
  - "Inspect how this was made" expands the trace tree for this one clip.

**Storage:**
- MP4 at `gs://whatif-prod/sessions/{ss}/clips/{cl}.mp4`, public-readable (or signed-URL gated).
- HLS variant at `…/clips/{cl}/hls/index.m3u8` for streaming on slow links.
- A static HTML at `…/c/{short_code}/index.html` pre-rendered server-side with OG tags so the link unfurls nicely.

## 2. Director's Cut

The end-of-session reel. One-click "Director's Cut" button assembles every pinned clip into a single composed MP4 plus the public landing page.

**What ships:**
- The composed MP4. Default cut order: chronological by source pts. User can reorder.
- Title card showing match teams + score + date + participant names.
- Per-clip branch cards as 1-second overlays.
- Outro card with credits and a link to the trace.
- A landing page at `https://whatif.app/cut/<short_code>`:
  - Hero player.
  - Chip strip of clips; selecting a chip seeks.
  - For each clip: prompt, source moment, validator verdict.
  - Full trace explorer accessible.

**Storage:**
- Final MP4 at `…/composed/{artifact_id}.mp4`.
- All referenced clips remain available as their own MP4s.

## 3. Provenance export bundle

A downloadable ZIP for judges, archivists, or anyone who wants the receipts.

**What ships:**
- Full `trace_event` rows for the artifact's full ancestry as `trace.jsonl`.
- All referenced blobs: reference frames, prompt bodies, model outputs, generated clips, validator outputs.
- A `manifest.json` mapping every trace ID → blob path inside the ZIP.
- A standalone HTML viewer (`viewer.html`) inside the ZIP that renders the trace tree locally without a server, by reading `manifest.json` and the bundled blobs. Open it in a browser and explore offline.

**Endpoint:** `POST /api/sessions/{id}/artifacts/{ar}/export` → returns a signed URL to the prepared ZIP. ZIP is built lazily on first request, cached after.

## Short codes

- 8-character URL-safe ULID prefix (`base32-crockford`, ambiguity-safe).
- Unique within the `artifact` table.
- Collision strategy: regenerate on collision (probability negligible at expected scale).
- Revocation: setting `artifact.revoked_at` immediately returns 410 Gone at the public route.

## OG / unfurling

Each share landing page is server-rendered with:
- `og:title` — match line + user's prompt.
- `og:description` — one-sentence summary auto-generated from the clip.
- `og:image` — a 1200×630 PNG poster auto-generated by Imagen 4 from the clip's first frame plus a title overlay (the validator's caption is the subtitle).
- `og:video` — direct MP4 for inline playback in apps that support it.

## Watermarking

- Veo outputs carry SynthID watermarking by default. We do not remove it.
- An explicit visible "What If" wordmark in the bottom-right of every generated clip and every composed reel. Wordmark is added by the compositor (ffmpeg `drawtext` + watermark PNG).

## Embed

`https://whatif.app/embed/c/<short_code>` returns an iframe-safe player with no chrome other than the bottom-right wordmark. Suitable for embedding in Substack, blog posts, or Discord rich previews.

## API

```
POST /api/sessions/{id}/artifacts          # create a new artifact of kind X
GET  /api/artifacts/{ar}                   # metadata
POST /api/artifacts/{ar}/reorder           # reorder clips for a director's cut
POST /api/artifacts/{ar}/regenerate        # rebuild the artifact (after a clip change)
POST /api/artifacts/{ar}/export            # prepare a provenance ZIP
DELETE /api/artifacts/{ar}                 # mark revoked (does not delete blobs immediately)
```

Public read routes:
```
GET /c/{short_code}          # single clip
GET /cut/{short_code}        # director's cut
GET /embed/c/{short_code}    # iframe player
```
